Вариант соответствует варианту лабораторной 4 – это задача, решаемая обслуживающим устройством №2 при помощи k (или k-3) вспомогательных потоков (k определено в лабораторной 4, далее л/р 4)
Если в л/р 4 реализация последовательного алгоритма выполняла расчет быстрее параллельной, доработать параллельный алгоритм, обосновать повышение производительности расчета, и только потом модифицированным алгоритмом решать задачу на обслуживавющем устройстве (ОУ) №2

Заявка содержит имя файла с графом и значения параметров задачи согласно варианту из л/р 4 при наличии оных
Перед пуском конвеера генератор помещает в 1ю очередь Q1 N штук заявок. N будет учавствовать в исследовании зависимости времени решения:
1. реализованным конвейером N задач (обработка N заявок)
2. последовательного решения N задач (в программе существует единственный поток без конвейера)

Схема конвейера

<генератор> -> <очередь Q1> -> <ОУ 1: чтение графа в заявку из файла, имя которого указано в заявке> -> <очередь Q2, мьютекс mQ2> -> <ОУ 2: решает задачу по варианту используя [k-3...k] вспомогательных потоков. здесь использовать наработки л/р 4> -> <очередь Q3, мьютекс mQ3> -> <ОУ 3: запись ответа в новый файл, полученный добавлением постфикса к имени исходного файла с графом> -> <пул обработанных заявок>

запись в соотвествующее поле заявки отмечается временем начала/окончания обработки заявки в iм ОУ

Ни одно ОУ ничего не пишет в консоль (только если 3 разных консоли). Когда все заявки обработаны (выполнены все стадии обработки всех заявок), единственный к этому времени поток программы формирует лог из 6*N событий, аннотированных меткой времени, типом события, № заявки и ID ОУ. 6N строковых описаний событий сортировать по возрастанию времени события.
